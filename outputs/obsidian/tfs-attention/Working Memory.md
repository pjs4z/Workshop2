# Working Memory

**Definition:** A limited-capacity cognitive system utilized to temporarily hold and manipulate multiple distinct ideas, digits, or operations simultaneously, especially under time pressure.

## Systems Dynamics & Relationships
[[Working Memory]] acts as the central stock of cognitive capacity. It is populated by a [[Task Set]] via [[Executive Control]]. Because items in [[Working Memory]] decay rapidly, it heavily influences the pace of thought, driving up [[Mental Effort]]. If [[Working Memory]] becomes overloaded, it triggers protective mechanisms like [[Attention Allocation]] and [[Inattentional Blindness]].
- **Upstream (Driven by):** [[Task Set]], [[Executive Control]]
- **Downstream (Drives):** [[Mental Effort]]

## Behavioral Engineering Context (COM-B)
[[Working Memory]] capacity dictates an individual's Capability to perform complex [[System 2]] operations.
- **Capability:** A rigid bottleneck; humans can only hold a few items in memory simultaneously before performance degrades.
- **Leverage Points:** Modify Opportunity (Environmental Design). Completely bypass [[Working Memory]] limitations by providing physical artifacts (e.g., paper, digital scratchpads) to store intermediate results. Eliminating artificial pacing (like strict deadlines) also prevents the rapid decay of information, keeping the system stable.
